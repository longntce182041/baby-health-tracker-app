API list and sample JSON

Base URL
- http://localhost:3000/api

Auth
- Use header: Authorization: Bearer <token>

1) Register parent
POST /register
{
  "email": "", //your email
  "password": "123456"
}

2) Verify OTP
POST /verify-otp
{
  "email": "", //your email
  "otp": ""   //otp form email
}

3) Login (returns token)
POST /login
{
  "email": "",//your email
  "password": "123456"
}

4) Forgot password (send OTP)
POST /forgot-password
{
  "email": ""//your email
}

5) Reset password
POST /reset-password
{
  "email": "parent1@example.com",
  "otp": "12345",
  "new_password": "NewP@ssw0rd123"
}

          },
          {

          }
        ],
        "note": "Available for consultation"
      },
      {
        "date": "2026-03-03T00:00:00.000Z",
        "slots": [
          {

            "end_time": "15:00"
          }

  Note: Doctor registers their available time slots for the week.

  27) Schedule consultation (auth required, parent role)
  POST /consultations
  {
    "doctor_id": "<doctor_id>",
    "baby_id": "<baby_id>",
    "date": "2026-03-01T00:00:00.000Z",
    "start_time": "08:00",
    "end_time": "09:00",
    "notes": "Short description of the issue"
  }
  Note: A conversation will be automatically created between parent and doctor.
  The notes will be sent as the first message from parent to doctor.
  Maximum 3 patients can be booked for the same time slot.
  Error if slot is fully booked or parent already has a booking for that slot.

  28) View consultation doctor list (auth required, parent role)
  GET /consultation-doctors

  28.5) View doctor schedule (auth required, parent role)
  GET /doctors/{doctor_id}/schedule
  GET /doctors/{doctor_id}/schedule?from_date=2026-03-01T00:00:00.000Z&to_date=2026-03-31T23:59:59.999Z

  Parameters:
  - doctor_id: (required) Doctor's ID
  - from_date: (optional) Start date for schedule range (default: today)
  - to_date: (optional) End date for schedule range (default: 30 days from today)

  Response:
  {
    "message": "Doctor schedule fetched",
    "doctor": {
      "_id": "<doctor_id>",
      "full_name": "Dr. Nguyen Van B",
      "specialty": "Pediatrics"
    },
    "data": [
      {
        "date": "2026-03-02T00:00:00.000Z",
        "status": "available",
        "note": "Available for consultation",
        "slots": [
          {
            "start_time": "08:00",
            "end_time": "09:00",
            "booked_count": 2,
            "available_slots": 1,
            "is_fully_booked": false,
            "is_user_booked": false
          },
          {
            "start_time": "09:00",
            "end_time": "10:00",
            "booked_count": 3,
            "available_slots": 0,
            "is_fully_booked": true,
            "is_user_booked": true
          }
        ]
      },
      {
        "date": "2026-03-03T00:00:00.000Z",
        "status": "available",
        "note": "",
        "slots": [
          {
            "start_time": "14:00",
            "end_time": "15:00",
            "booked_count": 0,
            "available_slots": 3,
            "is_fully_booked": false,
            "is_user_booked": false
          }
        ]
      }
    ]
  }

  Note: Each time slot can accommodate a maximum of 3 patients.
        - booked_count: Number of patients already booked for this slot
        - available_slots: Number of remaining slots (max 3)
        - is_fully_booked: True if slot already has 3 patients
        - is_user_booked: True if the current user is already booked for this slot

  29) Chat with doctor - send message (auth required, parent or doctor role)
  POST /conversations/send

  If Parent (use consultation_id or doctor_id):
  {
    "consultation_id": "<consultation_id>",
    "content": "Hello doctor, I have a question about my baby."
  }

  If Doctor (use consultation_id or parent_id):
  {
    "consultation_id": "<consultation_id>",
    "content": "Hi, I received your message. Let me help you."
  }

  Note: baby_id is no longer required. It is automatically retrieved from conversation/consultation.
        consultation_id is recommended for accurate conversation lookup.

  30) Chat with doctor - view conversation (auth required, parent or doctor role)
  GET /conversations?consultation_id=<consultation_id>
  OR
  GET /conversations?doctor_id=<doctor_id>

  Note: consultation_id is recommended for accurate conversation lookup.
        If using doctor_id, it will return the most recent conversation with that doctor.
